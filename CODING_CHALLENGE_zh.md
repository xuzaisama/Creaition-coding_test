# 实习生编程挑战：智能任务管理系统

## 概述
构建一个带有**智能功能**的**任务管理系统**，以展示你在不同技术方向上的编码能力。这个挑战旨在评估候选人在后端、前端、全栈以及 AI/LLM 开发等不同岗位上的综合表现。

**时间限制：** 2-4 小时  
**提交形式：** Git 仓库，并附带清晰的 README 与提交历史

---

## 题目说明

创建一个任务管理系统，使用户能够在智能辅助下创建、组织和管理任务。系统应包含一个后端 API，并可选择性地包含前端界面和/或 AI 驱动功能。

---

## 核心要求（所有候选人都需完成）

### 1. 任务 CRUD 操作
实现一个 RESTful API 或服务层，包含以下接口：

- **CREATE** - 新增任务
- **READ** - 根据 ID 获取任务，或获取全部任务列表
- **UPDATE** - 修改任务详情
- **DELETE** - 删除任务

### 2. 任务属性
每个任务应包含：
- `id`（唯一标识）
- `title`（字符串，必填）
- `description`（字符串，可选）
- `status`（枚举：`"pending"`、`"in_progress"`、`"completed"`）
- `priority`（枚举：`"low"`、`"medium"`、`"high"`）
- `created_at`（时间戳）
- `updated_at`（时间戳）
- `tags`（字符串数组，可选）

### 3. 数据持久化
请选择合适的存储方案：
- **简单方案：** JSON 文件或 SQLite
- **推荐方案：** MySQL、PostgreSQL、MongoDB 或 Redis
- **进阶方案：** 数据库 + 缓存层组合

### 4. 代码质量要求
- 代码整洁、可读性高、组织清晰
- 具备适当的错误处理
- 具备输入校验
- 至少提供基础文档（README）
- Git commit 信息清晰明确

## 岗位方向功能要求（根据目标岗位选择）

### 选项 A：后端开发方向

请实现以下高级后端功能：

1. **任务筛选与排序**
   - 支持按状态、优先级、标签筛选
   - 支持按创建时间、优先级、状态排序
   - 支持分页

2. **任务依赖关系**
   - 任务可以依赖其他任务
   - 若依赖任务尚未完成，则该任务不能被标记为完成
   - 提供查询任务依赖树的 API

3. **性能优化**
   - 为高频访问数据实现缓存
   - 优化数据库查询
   - 基础操作的 API 响应时间应小于 100ms

4. **系统设计**
   - 考虑系统扩展到 10 万以上任务时的设计方案
   - 附带一份简要架构文档

**加分项：**
- 实现搜索功能（全文检索）
- 增加用户认证 / 授权
- 为 API 接口添加限流
- 使用容器化（Docker）

---

### 选项 B：前端开发方向

构建一个具备以下功能的用户界面：

1. **任务仪表盘**
   - 以有组织的方式展示所有任务（列表 / 看板 / 日历）
   - 用可视化方式标识优先级和状态
   - 响应式设计（适配移动端）

2. **交互式任务管理**
   - 支持拖拽修改状态或调整顺序
   - 支持快捷操作（标记完成、删除、编辑）
   - 支持实时搜索 / 筛选

3. **用户体验**
   - 流畅的动画和过渡效果
   - 加载状态与错误提示
   - 表单校验并提供有帮助的反馈

4. **状态管理**
   - 使用合理的状态管理方案（Context / Redux / Vuex / Pinia）
   - 支持乐观更新
   - 优雅处理边界情况

**技术栈可选：**
- React + TypeScript + Tailwind / MUI
- Vue 3 + TypeScript + Element Plus
- 原生 JavaScript（如果你想展示基础能力）

**加分项：**
- 支持深色模式
- 键盘快捷键
- 离线支持（PWA）
- 数据可视化（图表 / 统计图）

---

### 选项 C：全栈开发方向

结合后端与前端要求：

1. **完整应用**
   - 可运行的后端 API（从后端方向功能中选择）
   - 可用的前端界面（从前端方向功能中选择）
   - 前后端之间集成完整、正确

2. **系统架构**
   - 清晰的关注点分离
   - RESTful API 设计或 GraphQL
   - 前后端都具备错误处理

3. **部署考虑**
   - 提供 Docker Compose 配置，或部署说明
   - 管理环境配置
   - 基础 CI/CD 配置（可选，但会加分）

**加分项：**
- 使用 WebSocket 实现实时更新
- 支持任务附件上传
- 支持任务导出（JSON / CSV / PDF）

---

### 选项 D：AI/LLM 开发方向

使用 AI / ML / LLM 构建智能功能：

1. **智能任务生成**
   - 支持自然语言创建任务，例如：`"Remind me to buy groceries tomorrow at 3pm"`
   - 从自然语言中提取：标题、描述、截止时间、优先级

2. **智能任务组织**
   - 根据任务内容自动推荐标签
   - 使用简单的 ML 或 LLM 进行优先级推荐
   - 任务自动分类

3. **AI 助手功能**（至少选择 2 项）
   - 任务拆解：将复杂任务拆分为子任务
   - 任务摘要：生成每日 / 每周任务摘要
   - 相似任务检测：查找相关任务
   - 智能搜索：使用语义搜索替代关键词匹配

4. **实现方式可选**
   - 使用 OpenAI API / Anthropic API（需说明 API Key 的处理方式）
   - 使用本地模型（HuggingFace、LangChain）
   - 使用 embedding + 向量数据库（ChromaDB、FAISS）
   - 构建简单的 ML 分类器（sklearn、PyTorch）

**加分项：**
- 为任务知识库实现 RAG
- 多智能体系统（规划 Agent + 执行 Agent）
- 针对任务领域进行模型微调
- Prompt 优化与测试

## 常见问题

**问：我可以使用外部库 / 框架吗？**  
答：可以，但请说明你的选择理由。使用过多不必要的库可能会影响评价。

**问：我可以使用 AI 助手（ChatGPT、Copilot）吗？**  
答：可以，但如果使用了，请在 README 中说明。我们更看重你的理解能力和决策过程，而不仅仅是“完美代码”。

**问：如果我没法把所有内容都做完怎么办？**  
答：提交你已完成的部分，并清楚说明哪些已完成、哪些仍待完成。质量比数量更重要。

**问：我可以同时选择多个方向吗？**  
答：可以。进阶候选人可以组合多个方向（例如全栈 + AI 功能），但前提是不要因此牺牲整体质量。

**问：我需要部署应用吗？**  
答：不是必须，但如果你能提供在线演示会很加分。至少请提供清晰的本地运行说明。
